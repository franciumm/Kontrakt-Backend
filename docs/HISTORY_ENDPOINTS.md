# History Endpoints Documentation

This document outlines the usage and expected responses for the `/history` endpoints in the Kontrakt API. These endpoints return paginated lists of historical data for the authenticated user.

---

## 1. Audit History

**Endpoint:** `GET /api/audit/history`  
**Auth Required:** Yes (JWT access token via `Authorization: Bearer <token>`)

Retrieves a paginated list of past contract audits generated by the user.

### Query Parameters

| Parameter | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `page` | Integer | `1` | The page number to retrieve (minimum `1`). |
| `limit` | Integer | `20` | The number of records per page (minimum `1`, maximum `100`). |

### Response Example

```json
{
  "success": true,
  "data": [
    {
      "_id": "64bfa1e72a8...",
      "userId": "64bfa1e72a8...",
      "originalText": "This is the original text of the contract that was audited...",
      "flags": [
        {
          "_id": "64bfa1e72a8...",
          "category": "work-for-hire-trap",
          "severity": "red",
          "clause_quote": "Client retains all rights to any and all works produced...",
          "plain_english": "The client owns everything you create right away."
        }
      ],
      "createdAt": "2026-07-11T12:00:00.000Z",
      "updatedAt": "2026-07-11T12:00:00.000Z",
      "__v": 0
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 45,
    "pages": 3
  }
}
```

## 2. Contract History

**Endpoint:** `GET /api/contract/history`  
**Auth Required:** Yes (JWT access token via `Authorization: Bearer <token>`)

Retrieves a paginated list of past contract interrogator sessions generated by the user.

> [!NOTE]
> To save bandwidth, the `generatedText` and `exposureReport` fields are **excluded** from this list view. If you need the full contract content or the detailed exposure report, you should call the single-contract endpoint `GET /api/contract/:id`.

### Query Parameters

| Parameter | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `page` | Integer | `1` | The page number to retrieve (minimum `1`). |
| `limit` | Integer | `20` | The number of records per page (minimum `1`, maximum `100`). |

### Response Example

```json
{
  "success": true,
  "data": [
    {
      "_id": "64bfb2f83b9...",
      "userId": "64bfa1e72a8...",
      "title": "Logo design for local coffee shop",
      "gigDescription": "Logo design for local coffee shop, $800",
      "gigType": "design",
      "answeredState": {
        "paymentTerms": "net-30",
        "revisions": "2"
      },
      "exposureScore": 85,
      "status": "finalized",
      "createdAt": "2026-07-11T12:05:00.000Z",
      "updatedAt": "2026-07-11T12:10:00.000Z",
      "__v": 0
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 12,
    "pages": 1
  }
}
```

## Error Responses

If authentication is missing, the standard error response is:

```json
{
  "success": false,
  "error": {
    "message": "Authentication required",
    "code": "AUTH_REQUIRED"
  }
}
```

If the authentication token is invalid or expired, the error response is:

```json
{
  "success": false,
  "error": {
    "message": "Invalid or expired access token",
    "code": "AUTH_INVALID"
  }
}
```
